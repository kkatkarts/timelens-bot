
📄 TimeLens: Project Brief

Аналитический пет-проект: Telegram-бот для анализа времени по категориям (iCloud Calendar)



🎯 1. Идея проекта

TimeLens — сервис, который подключается к календарю macOS (iCloud/CalDAV), агрегирует события по меткам и категориям, и отдаёт статистику в удобном формате. 

Пользовательский сценарий: 
«Сколько времени я потратила на спорт/учёбу/работу за неделю?» → Бот отвечает графиком и цифрами.

Формат: 
Telegram-бот (MVP) → при масштабировании: веб-дашборд. Ключевая фишка: анализ без ручного тайм-трекинга — данные уже есть в календаре.



💡 2. Обоснование ценности

2.1 Для пользователя (продукт)

Потребность / Решение проекта:

- «Хочу понимать, на что уходит время, но не хочу вести отдельный трекер» / Использует уже заполненный календарь — 0 дополнительных действий
- «Нужна наглядная статистика, как в банковском приложении» / Группировка по категориям, графики, сравнение периодов
«Хочу задавать вопросы просто, без фильтров и дат»
- NLP-интерфейс: «Покажи спорт за март» → готовый отчёт
- «Важна приватность» / Все данные обрабатываются локально / в контролируемом окружении


2.2 Для аналитика (портфолио)

Навыки и компетенции / Как демострируются в проекте:

- Работа с реальными данными
Парсинг CalDAV, очистка, обработка часовых поясов, RRULE, дедупликация
- Продуктовое мышление
От идеи → к юзер-флоу бота → к метрикам ценности
ETL / аналитический пайплайн
Выгрузка → трансформация → агрегация → визуализация → доставка
- Интеграция ИИ
Прагматичное применение LLM: категоризация, NL-интерфейс, генерация выжимок + fallback
Инфраструктурная грамотность
Конфигурация через .env, Docker, деплой, мониторинг — даже в MVP
- Коммуникация результата
Документация, демо-видео, читаемый README — артефакты для собеседований

🔑 Итог: Проект показывает не «знаю pandas», а «умею доводить аналитическую задачу до работающего продукта».
